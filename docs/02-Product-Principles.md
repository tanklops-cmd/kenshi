# 02 - Product Principles

These principles are mandatory. If a feature conflicts with these, the feature should be redesigned or moved to the backlog.

## Principles

1. **Reflection First**  
   Reflection is the heart of the application.

2. **Prepare the Next Keiko**  
   Every useful feature should help the user walk into the next keiko with more clarity.

3. **Software First**  
   The product must be excellent without AI. AI may be added later only as optional assistance.

4. **Offline First**  
   Core use must work without internet.

5. **No Account Required Initially**  
   The user can start using the app immediately. Sync and backup are optional later.

6. **User Owns Their Data**  
   Local-first data storage. Export and backup should be respected long term.

7. **One Purpose Per Screen**  
   Avoid clutter. Every screen must answer one primary question.

8. **Simple Beats Clever**  
   Prefer maintainable, predictable software over impressive complexity.

9. **Never Punish the User**  
   No guilt, streak loss, nagging, or shame for missed keiko or missed reflections.

10. **No Gamification**  
   No XP, streaks, badges, leaderboards, or achievement spam.

11. **Personal Knowledge Before Reference Knowledge**  
   When improving a topic, the user's own reflections, guidance, moments, and cues come before static reference content.

12. **The App Is Patient**  
   Martial arts improvement is not linear. The app must support long-term, uneven progress.

13. **One Moment = One Lesson**  
   Video Moments should be short, meaningful, and linked to learning.

14. **Build for Ten Years**  
   Prioritise maintainability, data safety, and clarity over speed.
